Release notes

Current stable version : 2.1.0 (2014-01-20)

This release adds the following new features:

  • Added a new module to process data from a database using JDBC API
  • Added a new module to process data from an xml file using StAX and JAXB APIs
  • Added a new API to read data from an in-memory String data source
  • Added a new API to read data from the Standard Input
  • Added data source name to Easy Batch report
  • Added 10 new tutorials about all Easy Batch features (basic, intermediate and advanced)
  • Updated documentation and Javadoc

Please check the change log for all details about changes.

Prior versions

Version : 2.0.0 (2014-01-12)

This is the first stable version of Easy Batch in the 2.x branch. This version brings a big refactoring to the former CB4J framework to make it able to process not only CSV input but also any type of input. This release adds many new features such as:

  • New Record interface for generic input data type
  • Record filtering through the RecordFilter API which allows to skip records from being processed by the engine
  • Redesigned validation approach which is now well integrated with the Bean Validation API (JSR303)
  • Simplified logging through a single and centralised log file
  • New EasyBatchReportFormatter API to format Easy Batch reports to a specific format (HTML, PDF, etc)
  • Updated documentation and Javadoc

Version : 1.4.0 (2013-09-21)

This release adds many new features such as:

  • New BatchResultHolder API to be able to get computation result at the end of batch execution
  • New CSV/Object mapper implementation to map records to domain objects based on field names declared in header record
  • New parameter to abort execution on first record processing error / record mapping exception
  • New parameter to enable/disable JMX monitoring feature
  • New validators to validate that date value is before/after a given date
  • New validators to validate that field field content size has a min/max length

Version 1.3.0 (2013-04-21)

This release adds the following new features:

  • Basic record processing transaction support
  • New JMX custom clients : command line and graphical asynchronous real time monitors
  • New batch configuration builder API

Version 1.2.0 (2013-01-13)

This release adds the following new features:

  • Fixed-Length Record parsing support
  • Quartz scheduling module
  • Hibernate integration use case
  • Spring integration use case

This release also includes some minor API changes and documentation updates. Please check the change log for all details about changes.

Version 1.1.0 (2012-12-08)

This release adds the following new features:

  • Support for data qualifier character
  • Support for whitespaces trimming
  • Support for execution abort on first reject
  • JMX monitoring support
  • More precise reporting about records processed with errors

For all details about changes in this release, please refer to the change log.

Version 1.0.2 (2012-12-01)

This is the second maintenance release in the 1.0.x branch. This release adds some minor changes to support loading configuration parameters file from the classpath. It also fixes a performance issue due to String concatenation. For all details about changes in this release, please refer to the change log.

Version 1.0.1 (2012-11-01)

This is the first maintenance release in the 1.0.x branch. The release includes multiple performance optimizations, bugfixes and documentation updates. The full list of changes may be found in the change log.

Version 1.0.0 (2012-10-14)

This is the first CB4J release. The main features of this release are :

  • Initial core API and Implementation
  • Common data type validators
